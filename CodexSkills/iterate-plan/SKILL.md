# Iterate Plan

Use this skill when a plan already exists and should be revised instead of rewritten from scratch.

## Workflow

1. Read the entire existing plan.
2. If the user feedback is incomplete, clarify what needs to change.
3. Research only the areas needed to validate the requested changes.
4. Present your understanding and approach before editing.
5. Make surgical updates while preserving structure unless the user wants a broader rewrite.
6. Keep file references accurate.
7. Preserve the separation between automated verification and manual verification.
8. Run `humanlayer thoughts sync` after the update.
9. Summarize what changed.

## Guardrails

- Be skeptical of vague requests.
- Read the whole plan before editing.
- Resolve open questions before finalizing the revision.
